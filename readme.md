# Modbus GUI 测试工具

一个基于 PySide6 的 Modbus 协议测试工具，支持 TCP 和 RTU 两种模式，提供批量测试和手动报文发送功能。

## 🎯 功能特点

### 核心功能
- ✅ 支持 **Modbus TCP** 和 **Modbus RTU** 协议
- ✅ 支持 **01/02/03/04/05/06/15/16** 功能码
- ✅ 支持 **BIT/UINT16/INT16/UINT32/INT32/FLOAT** 数据类型
- ✅ 支持 **ABCD/BADC/CDAB/DCBA** 字节顺序
- ✅ **批量测试**：Excel 驱动，自动化执行测试用例
- ✅ **手动发送**：自由构建和发送原始报文
- ✅ **实时日志**：彩色日志输出，支持级别过滤

### 数据解析
- ✅ BIT 类型按位解析，显示位名称
- ✅ 寄存器数据自动解析，显示寄存器描述
- ✅ 浮点数自动转换显示
- ✅ 响应报文解析（支持 TCP/RTU 两种格式）

### 用户体验
- ✅ 深色主题，护眼舒适
- ✅ 协议切换自动断开连接
- ✅ 批量测试异步执行，不卡 UI
- ✅ 配置自动保存（config.ini）
- ✅ 日志导出功能

## 🚀 安装与运行

### 1. 克隆项目

```bash
git clone https://github.com/pwq1995/modbus.git
cd modbus
```

### 2. 创建虚拟环境（推荐）


```bash
pip install -r requirements.txt
```

或手动安装：

```bash
pip install pyside6 pandas openpyxl pyserial crcmod numpy
```

### 4. 运行程序

```bash
python gui_main.py
```

## 📊 批量测试

### Excel 模板格式

| 列名 | 说明 | 示例 |
|------|------|------|
| 用例编号 | 测试用例编号 | 1 |
| 设备地址 | 从站 ID | 1 |
| 功能码 | Modbus 功能码 | 3 |
| 起始地址 | 寄存器起始地址 | 1626 |
| 寄存器数量 | 读取/写入数量 | 1 |
| 数据类型 | BIT/UINT16/INT16/UINT32/INT32/FLOAT | INT16 |
| 字节顺序 | ABCD/BADC/CDAB/DCBA | ABCD |
| 位名称 | BIT 类型的位名称（逗号分隔） | 柜门状态,柜门急停 |
| 寄存器描述 | 寄存器描述（逗号分隔） | 额定功率 |

### 支持的功能码

| 功能码 | 名称 | 类型 |
|--------|------|------|
| 01 | 读线圈 | 读取 |
| 02 | 读离散输入 | 读取 |
| 03 | 读保持寄存器 | 读取 |
| 04 | 读输入寄存器 | 读取 |
| 05 | 写单个线圈 | 写入 |
| 06 | 写单个寄存器 | 写入 |
| 15 | 写多个线圈 | 写入 |
| 16 | 写多个寄存器 | 写入 |

### 批量测试操作步骤

1. 点击 **"浏览"** 选择 Excel 文件
2. 点击 **"加载"** 加载测试用例
3. 点击 **"运行全部"** 执行所有用例
4. 或使用 **"上一步"/"下一步"** + **"执行当前行"** 单步调试

### BIT 类型位名称配置

对于 BIT 类型，在 Excel 的 "位名称" 列中按位顺序填写名称：

```
柜门状态,柜门急停,高压就绪,并网开关状态,户内道闸状态,备用,故障状态,高压电,运行状态,就绪状态,通信空置状态,备用,备用,备用,户外隔离开关主刀状态,户外隔离开关地刀状态
```

解析结果示例：
```
柜门状态=ON; 柜门急停=OFF; 高压就绪=ON; ...
```

## ✏️ 手动发送

### 从参数构建报文

1. 填写从站 ID、功能码、起始地址、数量/数据
2. 选择数据类型和字节顺序
3. 点击 **"从参数构建"** 生成报文
4. 点击 **"发送原始报文"** 发送

### 直接输入原始报文

在 "原始报文" 框中输入十六进制字节：

| 模式 | 输入格式 | 说明 |
|------|----------|------|
| RTU | `01 03 00 00 00 01` | 不含 CRC，自动计算并添加 |
| TCP | `01 03 00 00 00 01` | 不含 MBAP 头，自动添加 |

> **注意**：RTU 模式下，程序会自动计算 CRC16 校验值并附加到报文末尾。

### 写入数据格式

| 功能码 | 数据格式 | 示例 |
|--------|----------|------|
| 06（写单寄存器） | 单个整数 | `333` |
| 16（写多寄存器） | 逗号分隔的整数 | `100,200,300` |
| 16（写多寄存器，FLOAT） | 单个浮点数 | `3.14` |
| 15（写多线圈） | 逗号分隔的 0/1 | `1,0,1,1` |

## 🔍 数据解析

### 响应报文解析

发送报文后，点击 **"解析数据"** 按钮，程序会自动解析响应：

1. **错误响应识别**：自动识别并显示错误码含义
2. **BIT 类型**：按位展开，显示每个位的 ON/OFF 状态
3. **寄存器类型**：根据数据类型和字节顺序解析
4. **浮点数**：自动显示为浮点数格式

### 解析参数说明

| 参数 | 说明 | 选项 |
|------|------|------|
| 数据类型 | 数据解析格式 | BIT/UINT16/INT16/UINT32/INT32/FLOAT |
| 字节顺序 | 多字节数据的字节排列 | ABCD/BADC/CDAB/DCBA |
| 起始位/索引 | 从第几位开始解析 | 0-9999 |
| 数量 | 解析的位数或寄存器数 | 1-9999 |

## ⚙️ 配置文件

程序会自动生成 `config.ini` 配置文件：

```ini
[tcp]
host = 127.0.0.1
port = 502

[rtu]
port = COM1
baudrate = 9600

[timing]
timeout_sec = 3
```

### 配置项说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `tcp.host` | TCP 服务器 IP | 127.0.0.1 |
| `tcp.port` | TCP 端口 | 502 |
| `rtu.port` | RTU 串口号 | COM1 |
| `rtu.baudrate` | RTU 波特率 | 9600 |
| `timing.timeout_sec` | 超时时间（秒） | 3 |

## 📦 打包为 EXE

### 使用 PyInstaller + UPX（推荐）

1. **下载 UPX**：https://github.com/upx/upx/releases
2. **将 `upx.exe` 放到项目目录**
3. **运行打包命令**：

```bash
.venv\Scripts\activate
pip install pyinstaller
pyinstaller --onefile --windowed --upx-dir "." --exclude-module matplotlib --exclude-module scipy --exclude-module IPython --exclude-module jupyter --add-data "config.ini;." --add-data "1.xlsx;." --collect-all PySide6 --hidden-import numpy --hidden-import pandas --hidden-import openpyxl gui_main.py
```

### 使用打包批处理

项目提供了多种打包批处理文件：

| 文件 | 说明 |
|------|------|
| `build.bat` | 基础打包（不含 UPX） |
| `build_optimized.bat` | 优化打包（含 UPX 压缩） |
| `build_onedir.bat` | 文件夹模式打包 |
| `build_full.bat` | 完整打包（自动下载 UPX） |

运行方式：

```bash
.\build_full.bat
```

### 打包后文件结构

```
dist/
└── gui_main.exe          # 主程序
config.ini                 # 配置文件（需与 exe 同目录）
1.xlsx                     # Excel 模板（需与 exe 同目录）
```

## ❓ 常见问题

### 1. TCP 连接失败（10061）

- 检查目标设备 IP 和端口是否正确
- 确认目标设备已启动 Modbus TCP 服务
- 检查防火墙设置

### 2. RTU 连接失败

- 检查串口号是否正确（如 COM1）
- 确认串口未被其他程序占用
- 检查波特率、校验位等参数是否与设备匹配

### 3. Excel 加载失败

- 确保 Excel 文件包含必要的列
- 检查列名是否与模板一致
- 保存 Excel 文件时关闭其他程序

### 4. BIT 解析显示不完整

- 检查 "位名称" 列中的名称数量是否与寄存器数量匹配
- BIT 类型会全部显示，不会截断

### 5. 打包后 exe 无法运行

- 确保 `config.ini` 和 `1.xlsx` 与 exe 在同一目录
- 尝试以管理员身份运行
- 检查杀毒软件是否误报

## ⚠️ 注意事项

1. **RTU 报文输入**：手动发送时只需输入不含 CRC 的报文，程序会自动计算并添加
2. **TCP 报文输入**：手动发送时只需输入不含 MBAP 头的 PDU，程序会自动添加
3. **Excel 模板**：列名必须与模板一致，否则加载失败
4. **配置保存**：连接成功后自动保存配置，下次启动自动加载
5. **日志级别**：可在运行中切换日志级别过滤显示
